IDM 241 - Beta: Micro Interaction Redesign for Grammarly

Current State

Thoughts

Currently, the micro interaction is very static. There is no transition/easing into the main hover state.

This micro interaction contains a child micro interaction when hovering over the icons that appear after the initial hover.

-

I would like to bring an eased transitional quality as well as improve the indicative qualities using CSS.

Concept for redesign

Triggers

On-Hover, Off-Hover

Rules

On-Hover: Main Box

- First apply fog effect over box content and increase shadow intensity.

- Then move download and trash icons in from sides.

On-Hover: Download & Trash Icons

- Change Download Icon fill to Grammarly Green, Trash Icon to red.

On-Hover: Red # Tab

- Show alert description w/ shadow.

Off-Hover: Rewind the effects of on-hover.

Feedback

Visual: On-Hover - Main Boxes, ease in shadow increase and fog effect.

Visual: On-Hover - Download & Trash Icons, slide towards center from outside Main Box boundaries.

Visual: On-Hover - Download & Trash Icons, ease in green or red fill.

Visual: On-Hover - Red # Tab, fade in alert description bubble.

Loops & Modes

None.
